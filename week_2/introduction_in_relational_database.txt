Реляционная модель данных.
Структурна часть.
Типы данных - близки к ЯП (int, bigint, smallint, numeric/decimal,
float, char, varchar, date, time, datetime, time stamp, NULL)

Домен - логическое значение над элементом базового типа, которое
определяет подмножество значений типа данных (или моими словами
- возможные значение в ячейке).

Атрибут - записи об определенных параметрах каждой из сущностей
Например, для сущности "заказчик", видимо, будет храниться
информация об его наименовании, представителях, адресе и т.п.

Кортеж - множесто упорядоченных триплетов: имя атрибута, типа
данных или домен, значение.

describe name_project.name_table - посмотреть структуру таблицы

Схема отношений - именованное множество упорядоченных пар:
имя атрибута - тип данных (иди домен).

Схема базы данных - множество схем отношений (с возможными
ссыдками одной таблицы на другую).

Тело отношения - множество кортежей, соответствующих схеме
отноений (добавление и удаление кортежей)

Значение отношения - пара множеств, схема отношения, тело отношения.

ОГРАНИЧЕНИЕ ЦЕЛОСТНОСТИ.
Ограничение целостности сущности - все строки в таблице должны
хоть чем-нибудь отличаться. Виды ключей: простые (состоят из одного
атрибута), составные (состоят из нескольких атрибутов).
